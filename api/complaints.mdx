---
title: "Resolve complaint tickets with proof images"
description: "Submit a proof image to close a complaint ticket. HCIMS AI validates the upload with synthetic image detection before marking the ticket resolved."
---

The complaints endpoint handles the final step of the complaint resolution workflow: a staff member or administrator uploads a photograph proving the issue has been addressed. Before storing the image and marking the ticket resolved, HCIMS validates the proof using an AI model that detects synthetically generated or AI-fabricated images. If the image is flagged as synthetic, the upload is blocked, the ticket remains open, and the administrator receives a notification.

---

## POST /complaints/{complaint_id}/resolve_with_proof

Resolve an open complaint ticket by submitting a proof image. The request must be sent as `multipart/form-data`.

### Path parameters

<ParamField path="complaint_id" type="string" required>
  The UUID of the complaint ticket to resolve.
</ParamField>

### Request headers

<ParamField header="Authorization" type="string" required>
  `Bearer <supabase_access_token>` — must belong to an admin who owns the ticket, or a department staff member whose department matches the ticket's department.
</ParamField>

### Form fields

<ParamField body="proof_image" type="file" required>
  The proof photograph. Must have a `Content-Type` of `image/*` (JPEG, PNG, WebP, etc.). PDF files are not accepted. Maximum size is controlled by the `PROOF_IMAGE_MAX_BYTES` environment variable, which defaults to **10 MB**.
</ParamField>
<ParamField body="resolution_notes" type="string">
  Optional free-text notes describing how the issue was resolved. Saved to the ticket record when provided.
</ParamField>

### Image requirements

<Info>
  Only `image/*` content types are accepted. PDFs and other file types are rejected at `400` before any AI processing occurs.
</Info>

| Constraint | Value |
|---|---|
| Accepted MIME types | `image/*` (JPEG, PNG, WebP, GIF, etc.) |
| Maximum file size | `PROOF_IMAGE_MAX_BYTES` — default **10 MB** |
| Minimum file size | Non-empty (zero-byte files are rejected) |

### AI validation

After basic validation, HCIMS passes the image bytes to the `detect_synthetic_image` function. This model checks whether the image was generated by AI tools rather than captured by a real camera.

- If the image is determined to be synthetic, the endpoint returns `409` and the ticket remains open.
- A notification is inserted into the admin's notification feed with the staff member's name and the synthetic confidence score.
- If the model itself fails to load or is misconfigured, the endpoint returns `503`.

### Success response

<ResponseField name="success" type="boolean" required>
  `true` when the ticket was resolved and the proof image stored.
</ResponseField>
<ResponseField name="proof_url" type="string" required>
  Public Supabase Storage URL for the uploaded proof image.
</ResponseField>
<ResponseField name="message" type="string" required>
  Always `"Ticket resolved with a verified proof image."` on success.
</ResponseField>
<ResponseField name="ai_detection" type="object" required>
  Result from the synthetic image detection model.
  <Expandable title="ai_detection properties">
    <ResponseField name="is_synthetic" type="boolean" required>
      `true` if the model classified the image as synthetically generated.
    </ResponseField>
    <ResponseField name="confidence" type="number" required>
      Overall model confidence score between `0` and `1`.
    </ResponseField>
    <ResponseField name="authentic_confidence" type="number" required>
      Probability the image is a genuine photograph, between `0` and `1`.
    </ResponseField>
    <ResponseField name="synthetic_confidence" type="number" required>
      Probability the image is AI-generated, between `0` and `1`.
    </ResponseField>
    <ResponseField name="provider" type="string" required>
      Name of the detection model or provider used.
    </ResponseField>
    <ResponseField name="threshold" type="number" required>
      The `synthetic_confidence` value at or above which uploads are blocked.
    </ResponseField>
  </Expandable>
</ResponseField>

### Blocked synthetic image response (409)

When the image is flagged, the `detail` field of the `409` response is an **object**, not a string:

<ResponseField name="detail.message" type="string" required>
  Always `"AI-generated image detected. Upload a real photo to resolve this ticket."`.
</ResponseField>
<ResponseField name="detail.ai_detection" type="object" required>
  Same shape as the success `ai_detection` object above.
  <Expandable title="ai_detection properties">
    <ResponseField name="is_synthetic" type="boolean" required>
      `true` — the image was flagged as synthetic.
    </ResponseField>
    <ResponseField name="confidence" type="number" required>
      Overall model confidence.
    </ResponseField>
    <ResponseField name="authentic_confidence" type="number" required>
      Probability the image is genuine.
    </ResponseField>
    <ResponseField name="synthetic_confidence" type="number" required>
      Probability the image is AI-generated.
    </ResponseField>
    <ResponseField name="provider" type="string" required>
      Detection model or provider name.
    </ResponseField>
    <ResponseField name="threshold" type="number" required>
      The threshold value that triggered the block.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash Resolve with proof image
curl --request POST \
  --url http://localhost:8001/complaints/abc123/resolve_with_proof \
  --header 'Authorization: Bearer <supabase_access_token>' \
  --form 'proof_image=@/path/to/proof.jpg' \
  --form 'resolution_notes=Room thoroughly cleaned and towels replaced.'
```

```bash Minimal request (no notes)
curl --request POST \
  --url http://localhost:8001/complaints/abc123/resolve_with_proof \
  --header 'Authorization: Bearer <supabase_access_token>' \
  --form 'proof_image=@/path/to/proof.jpg'
```

### Errors

| Status | Detail |
|---|---|
| `400` | No filename provided, non-image file type, empty file, or file exceeds size limit |
| `401` | Missing or expired Supabase access token |
| `403` | Authenticated user is not authorised to resolve this ticket |
| `404` | No ticket found with the given `complaint_id` |
| `409` | Ticket is already resolved, or proof image was flagged as AI-generated |
| `503` | Supabase credentials not configured, or AI detection model unavailable |
